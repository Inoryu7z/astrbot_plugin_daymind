### v1.1.0
**🧩 多人格分桶与白名单机制上线**

**1. 🧠 思考 / 日记状态改为按人格隔离**
*   DayMind 核心运行状态从单份全局状态改为按人格分桶管理。
*   不同人格现在拥有各自独立的今日思考流、当前状态文本、日记生成状态与本地历史记录。
*   本地 `reflections/` 与 `diaries/` 存储已调整为按人格目录分层，避免不同人格内容混流。

**2. ✅ 新增人格白名单配置**
*   新增 `enabled_personas` 配置项。
*   只有命中白名单的人格才会启用 DayMind，未命中的人格会直接跳过。
*   思考与日记共用同一套白名单，避免 Bot 拥有大量人格时产生无意义调度压力。

**3. ⏱️ 新增思考调度随机抖动**
*   新增 `thinking_interval_jitter_seconds` 配置项。
*   自动思考在固定间隔基础上支持每轮加入随机抖动，用于错开并发峰值。
*   抖动按每轮重新计算，而不是固定偏移，适合多实例 / 多人格并行场景。

**4. 🎮 手动指令改为跟随当前人格**
*   `/手动思考`、`/生成日记`、`/清除今日思考` 均会基于当前会话解析人格。
*   若当前人格未命中白名单，会直接跳过并提示，不会偷偷生成内容。

**5. 📓 历史日记参考改为人格内读取**
*   日记生成时读取的历史日记不再是全局共享，而是只参考当前人格自己的历史日记目录。
*   这让同一个 Bot 下的不同人格可以保持各自连续的叙事线与生活感。

**6. ⚙️ AstrBot 配置界面字段补充**
*   已在 `_conf_schema.json` 中加入人格白名单与思考抖动配置字段。
*   用户可以在 AstrBot 提供的配置界面中直接管理这两项能力。

### v1.0.0
**🧠 首次发布：让 Bot 拥有“今天”的状态感**

**1. 🧠 自动思考链路上线**
*   支持按设定间隔自动生成当下思考，让 Bot 对“今天此刻正在经历什么”保持连续感。
*   思考会综合参考当前日程、最近对话、人格设定与最近思考内容。
*   已加入本地近似去重机制，减少重复表达与车轱辘式状态更新。

**2. 📓 自动日记能力上线**
*   支持在指定时间自动生成今日日记。
*   日记会融合今日日程、当日思考、人格设定与最近历史日记，形成更完整的一天记录。
*   已针对“第二人称污染”“像在对用户说话”这类问题进行了专门修正。

**3. 🔗 人格与现实信息接入**
*   支持解析当前会话人格名称与人格设定文本，让思考与日记更贴近既有人设气质。
*   可联动 `astrbot_plugin_life_scheduler` 获取天气、日程、穿着等现实轨迹，使内容更贴近“真实生活”。

**4. 🧠 LivingMemory 写入与复写保护**
*   支持将生成的日记写入 LivingMemory。
*   当允许重复生成今日日记时，旧 diary memory 不再物理删除，而是标记为 `deleted`，保留追踪痕迹并避免误删。

**5. 🧷 内容管理基础能力上线**
*   已支持日记星标与备注。
*   已支持按日聚合的思考流星标与备注。
*   被星标内容不会参与自动轮换删除。
*   支持分别管理日记与思考流的保留策略。

**6. 🌐 DayMind WebUI 基础完成**
*   已补齐 DayMind 自身 WebUI 所需的后端基础能力。
*   支持读取内容列表、查看详情、修改星标、修改备注、更新内部配置、清除今日思考流。
*   后续可继续独立迭代前端界面，而无需反向改动核心内容逻辑。

**7. 🧹 今日思考流清空能力上线**
*   新增 `/清除今日思考` 指令。
*   执行后会清空今日思考缓存、当前状态文本与本地当日 reflections 文件，使本日思考流回到空白状态。
