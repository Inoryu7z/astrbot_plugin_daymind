* 调度器休眠计算现在会加入 `silent_end_candidate`，确保在静默时段结束时及时醒来开始思考。

### v1.3.0

**🧠 架构整理：人格识别、会话绑定、自动补全与状态持久化全面升级**

**1. 🧩 人格识别升级为多别名匹配**

* 现在人格识别支持多别名匹配：`name` / `alias` / `persona_id` / `display_name` / `select_persona` ，插件会自动归一化成 canonical persona。

**2. 🔗 会话人格活跃映射自动管理**

* 可配置保留天数 `session_persona_retention_days` 和最大条目数 `session_persona_max_entries`。

**3. 🧠 思考 / 日记前可自动补生成今日日程**

* 新增 `reflection_auto_ensure_today_schedule` 开关。
* 开启后，思考或日记生成前如果检测到今日日程缺失，会自动调用 DayFlow 补生成。
* 避免日程空了导致思考没素材、日记生成失败的问题。
* 手动生成日记时，如果补生成日程，会在返回结果里明确告诉你“已自动补生成今日日程”。

**4. 🔁 思考与日记生成支持内部重试**

* 新增 `reflection_generation_retry_count` / `reflection_generation_retry_delay_seconds`。
* 新增 `diary_generation_retry_count` / `diary_generation_retry_delay_seconds`。
* 模型偶尔抽风或输出格式不对时，可以自动重试，提高成功率。

**5. 🌐 WebUI 支持按人格查询与管理**

* WebUI 接口新增 `persona_name` 参数，支持按人格筛选日记和思考流。
* 状态页返回 `available_personas` 和 `persona_status_map`，多人格管理一目了然。
* “清除今日思考”接口支持指定人格。

**6. 📓 新增 `/今日日记` 指令**

* 快速查看当前人格今天的日记内容。
* 别名：`查看今日日记` / `daymind_diary_today`。
* 和 `/生成日记` 的区别：只读不生成，如果今天还没日记会直接提示。

**7. 🔗 依赖管理重写，DayFlow 兼容更稳**

* `core/dependency.py` 重写，区分新版 `astrbot_plugin_dayflow_life_scheduler` 和旧版 `astrbot_plugin_life_scheduler`。
* 不只看插件名，还会检查实例接口是否完整，避免兼容性问题。
* 新增 `ensure_today_schedule()` 统一封装“缺日程时补生成”逻辑。

**8. 🧹 推送机制简化**

* 去掉了 `push_retry_times` 和 `push_retry_delay_seconds` 配置。

**10. 📝 默认思考 / 日记模板换代**

### v1.2.0 - 心情系统上线

**🧠 心情系统**上线，让 Bot 的对话风格能随心情动态变化。

**1. 💜 心情生成与注入**

* 每次思考后自动生成心情标签（如“平静”“烦躁”“疲惫”“开心”“专注”等）。
* 心情标签会影响下次思考前的对话风格：语气、回复长度、措辞等。
* 支持独立 Provider 模式（配置心情专用模型）和内联提取模式（从思考文本提取）。

**2. 🎮 新增心情相关指令**

* `/查看心情`：查看当前人格的心情状态和风格影响。
* `/今日心情`：查看当前人格今日的心情变化历史。

**3. ⚙️ 心情相关配置项**

* `enable_mood_system`：是否启用心情系统。
* `inject_mood_into_reply`：是否将心情注入对话风格。
* `mood_provider_id`：心情模型提供商（留空则从思考中提取）。
* `mood_reference_reflection_count`：心情提取时参考最近几条思考。
* `mood_max_history_per_day`：每日保留心情记录数。
* `mood_style_strength`：心情风格强度（弱 / 中 / 强）。
* `mood_allow_sharp_tone`：是否允许明显的尖锐语气。

### v1.1.0

**🧩 多人格分桶与白名单机制上线**

**1. 🧠 思考 / 日记状态改为按人格隔离**

* DayMind 核心运行状态从单份全局状态改为按人格分桶管理。
* 不同人格现在拥有各自独立的今日思考流、当前状态文本、日记生成状态与本地历史记录。
* 本地 `reflections/` 与 `diaries/` 存储已调整为按人格目录分层，避免不同人格内容混流。

**2. ✅ 新增人格白名单配置**

* 新增 `enabled_personas` 配置项。
* 只有命中白名单的人格才会启用 DayMind，未命中的人格会直接跳过。
* 思考与日记共用同一套白名单，避免 Bot 拥有大量人格时产生无意义调度压力。

**3. ⏱️ 新增思考调度随机抖动**

* 新增 `thinking_interval_jitter_seconds` 配置项。
* 自动思考在固定间隔基础上支持每轮加入随机抖动，用于错开并发峰值。
* 抖动按每轮重新计算，而不是固定偏移，适合多实例 / 多人格并行场景。

**4. 🎮 手动指令改为跟随当前人格**

* `/手动思考`、`/生成日记`、`/清除今日思考` 均会基于当前会话解析人格。
* 若当前人格未命中白名单，会直接跳过并提示，不会偷偷生成内容。

**5. 📓 历史日记参考改为人格内读取**

* 日记生成时读取的历史日记不再是全局共享，而是只参考当前人格自己的历史日记目录。
* 这让同一个 Bot 下的不同人格可以保持各自连续的叙事线与生活感。

**6. ⚙️ AstrBot 配置界面字段补充**

* 已在 `_conf_schema.json` 中加入人格白名单与思考抖动配置字段。
* 用户可以在 AstrBot 提供的配置界面中直接管理这两项能力。

### v1.0.0

**🧠 首次发布：让 Bot 拥有“今天”的状态感**

**1. 🧠 自动思考链路上线**

* 支持按设定间隔自动生成当下思考，让 Bot 对“今天此刻正在经历什么”保持连续感。
* 思考会综合参考当前日程、最近对话、人格设定与最近思考内容。
* 已加入本地近似去重机制，减少重复表达与车轱辘式状态更新。

**2. 📓 自动日记能力上线**

* 支持在指定时间自动生成今日日记。
* 日记会融合今日日程、当日思考、人格设定与最近历史日记，形成更完整的一天记录。
* 已针对“第二人称污染”“像在对用户说话”这类问题进行了专门修正。

**3. 🔗 人格与现实信息接入**

* 支持解析当前会话人格名称与人格设定文本，让思考与日记更贴近既有人设气质。
* 可联动 `astrbot_plugin_life_scheduler` 获取天气、日程、穿着等现实轨迹，使内容更贴近“真实生活”。

**4. 🧠 LivingMemory 写入与复写保护**

* 支持将生成的日记写入 LivingMemory。
* 当允许重复生成今日日记时，旧 diary memory 不再物理删除，而是标记为 `deleted`，保留追踪痕迹并避免误删。

**5. 🧷 内容管理基础能力上线**

* 已支持日记星标与备注。
* 已支持按日聚合的思考流星标与备注。
* 被星标内容不会参与自动轮换删除。
* 支持分别管理日记与思考流的保留策略。

**6. 🌐 DayMind WebUI 基础完成**

* 已补齐 DayMind 自身 WebUI 所需的后端基础能力。
* 支持读取内容列表、查看详情、修改星标、修改备注、更新内部配置、清除今日思考流。
* 后续可继续独立迭代前端界面，而无需反向改动核心内容逻辑。

**7. 🧹 今日思考流清空能力上线**

* 新增 `/清除今日思考` 指令。
* 执行后会清空今日思考缓存、当前状态文本与本地当日 reflections 文件，使本日思考流回到空白状态。
